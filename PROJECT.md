# Certificate Generator — Генератор сертификатов

**GitHub**: https://github.com/ElazAzel/certificate-generator

Полноценное веб-приложение для массовой генерации именных PDF-сертификатов.
Загрузите Excel с участниками, наложите шаблон, настройте поля — и получите готовые PDF.

---

## Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                      start.bat                          │
│  (один файл: установка → сборка → запуск → браузер)     │
└──────────┬──────────────────────────────────────────────┘
           │
    ┌──────┴──────┐
    │  Express    │  ← сервер (:3001)
    │  (server/)  │
    ├─────────────┤
    │ pdfService  │  ← PDF-lib + fontkit (кириллица)
    │ fontService │  ← сканирование C:\Windows\Fonts
    │ excelService│  ← xlsx парсинг
    └──────┬──────┘
           │
    ┌──────┴──────┐
    │ React+Vite  │  ← клиент, отдаётся статикой
    │ (client/)   │     (или dev-режим с прокси)
    └─────────────┘
```

**Стек**: TypeScript, React 18, Vite 5, Express 4, pdf-lib, pdfjs-dist 6, xlsx, fontkit, archiver.

## Полный функционал

### 1. Загрузка данных (Excel)
- Форматы: `.xlsx`, `.xls` (через библиотеку `xlsx`)
- Автоматически определяются колонки и все строки
- Предпросмотр таблицы с навигацией по строкам
- Скачивание шаблонного Excel-файла (кнопка "Скачать шаблонный Excel-файл")

### 2. Загрузка шаблона
- Поддержка PNG, JPG/JPEG, PDF
- Автоматическое определение размеров (ширина × высота в PT)
- PDF-шаблоны рендерятся через `pdfjs-dist` (первая страница → canvas)
- **По умолчанию**: при старте сервера автоматически загружается `template/шаблон.pdf`

### 3. Визуальный редактор полей
- Поля текста накладываются прямо поверх шаблона
- **Перетаскивание** (drag & drop) мышью
- **Изменение размера** через 4 угловых маркера (resize handles)
- **Масштаб**: 50%–150% + кнопка "По размеру экрана"
- ResizeObserver для автофита при ресайзе окна

### 4. Настройка каждого поля
- **Привязка к колонке Excel** (выпадающий список)
- **Шрифт**: все системные шрифты Windows + загруженные пользователем TTF/OTF
- **Размер шрифта** (pt)
- **Цвет текста**: color picker + hex ввод с валидацией
- **Выравнивание**: горизонтальное (left/center/right) и вертикальное (top/middle/bottom)
- **Поворот** (градусы, -360…360)
- **Режим переполнения**:
  - `single-line` — одна строка, выход за границы
  - `shrink-to-fit` — автоуменьшение шрифта до вписывания
  - `multiline` — перенос строк с заданным `lineHeight`
  - `clip` — обрезание текста с многоточием
- **Жирный / Курсив**
- **Видимость** (чекбокс — скрытые поля не генерируются)
- **Дублирование** поля со сдвигом +20px
- **Удаление** с подтверждением

### 5. Экспорт и генерация
- **Режимы**:
  - Раздельные PDF-файлы (каждый сертификат отдельно)
  - Один общий PDF (все сертификаты в одном файле)
- **Шаблон имени файла**: `{name}.pdf`, `{name}_{id}.pdf` — подстановка значений из колонок
- **Папка сохранения** на сервере
- **Тестовый PDF**: быстрая генерация для активной строки (открывается в новой вкладке)
- **Скачивание**: ZIP-архив для режима "раздельные" или единый PDF
- **Прогресс-бар** с анимацией shimmer
- **Модальное окно** результата: статистика (всего/успешно/ошибки), список ошибок

### 6. Сохранение и загрузка конфигурации
- **Экспорт JSON**: сохраняет layout полей + настройки экспорта
- **Импорт JSON**: восстанавливает конфигурацию полей
- **Сброс**: очистка всех данных (с подтверждением)

### 7. Шрифты
- **Системные**: при старте сканируется `C:\Windows\Fonts` — до 300+ шрифтов
- **Загрузка пользовательских**: TTF/OTF через интерфейс
- **Парсинг имени**: чтение name-таблицы (Name ID 1) из TTF/OTF бинарника
- **Встраивание в PDF**: через `@pdf-lib/fontkit` для корректной кириллицы
- **Резервный Arial**: `server/fonts/arial.ttf` вшит в репозиторий

### 8. UI/UX особенности
- **Glassmorphism**: полупрозрачные панели с `backdrop-filter: blur(16px)`
- **Фоновый градиент**: цветные орбы (indigo/violet/green)
- **Тёмная тема**: система переменных CSS, готовность к тёмной теме
- **Анимации**: modal scaleIn, toast slideUp, shimmer прогресс-бара, hover-эффекты
- **prefers-reduced-motion**: все анимации отключаются для accessibility
- **Фокус-ловушка**: в модалке Tab циклится между кнопками
- **Клавиатурная доступность**: upload-box реагирует на Enter/Space
- **Кастомные скроллбары** (WebKit + Firefox)
- **Адаптивность**: 3 брейкпоинта (1100px / 900px / 600px)
- **Тосты**: неблокирующие уведомления (success 3s, error 5s)

### 9. Безопасность
- Path traversal защита: `isSafePath()` проверяет, что путь внутри проекта
- Санитизация имён файлов: удаление недопустимых символов Windows/Linux
- `cleanUploads()`: очистка временных файлов при старте
- Multer: ограничение размера загрузки 50MB

## API Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/upload/excel` | Загрузка Excel-файла |
| POST | `/api/upload/template` | Загрузка шаблона (PNG/JPG/PDF) |
| POST | `/api/upload/font` | Загрузка шрифта (TTF/OTF) |
| GET | `/api/upload/fonts` | Список всех доступных шрифтов |
| GET | `/api/upload/templates` | Список зарегистрированных шаблонов |
| POST | `/api/generate` | Пакетная генерация сертификатов |
| POST | `/api/generate/test` | Тестовая генерация для одной строки |
| GET | `/api/download/:exportId` | Скачивание результатов (?type=zip или ?type=pdf) |
| GET | `/api/download/excel-template` | Скачивание шаблонного Excel |
| GET | `/api/health` | Health-check |

## Запуск

### Быстрый старт (рекомендуется)
```
start.bat
```
Установит зависимости → соберёт клиент → запустит сервер → откроет браузер.

### Вручную
```bash
cd server && npm install && npx tsx src/index.ts   # Сервер :3001
cd client && npm install && npm run dev             # Клиент :5173 (с прокси)
```

### Режим разработки
```bash
npm run dev      # concurrently запускает сервер+клиент
```

## Технические детали

### PDF-генерация
- `pdf-lib` + `@pdf-lib/fontkit` для встраивания шрифтов
- Координаты UI→PDF: инверсия Y (top-left → bottom-left)
- Поддержка всех режимов переполнения текста
- Бинарный поиск (`shrinkTextToFit`) для автоуменьшения
- Объединение PDF: `copyPages` из исходного PDF-шаблона
- Для изображений: `embedPng` / `embedJpg` и `drawImage`

### Системные шрифты
- Сканирование `C:\Windows\Fonts` (TTF + OTF)
- Парсинг name-таблицы OpenType/TrueType (Name ID 1 = Font Family)
- Отображение реальных имён шрифтов в выпадающем списке
- Реестр шрифтов: in-memory Map, id → info

### Работа с PDF-шаблонами
- pdfjs-dist v6 для рендера превью первой страницы
- Worker загружается локально (не CDN)
- Canvas 1.5× scale для качества
- Авто-регистрация `template/шаблон.pdf` при старте сервера

### UX фичи
- Glassmorphism через backdrop-filter
- Анимации с prefers-reduced-motion
- Фокус-ловушка и автофокус в модалках
- Toast-уведомления (не alert)
- Адаптив: 3 брейкпоинта
- ErrorBoundary для отлова ошибок рендера

## Известные ограничения

1. **Прогресс бар**: генерация одним вызовом API — прогресс прыгает 0→100%. Для реального прогресса нужны SSE/WebSocket.
2. **PDF worker**: 1.2MB — влияет на время первой загрузки, но обеспечивает offline-работу.
3. **Только Windows шрифты**: при старте сканируется `C:\Windows\Fonts`. На Linux/Mac потребуется другая директория.
4. **In-memory registry**: шаблоны и шрифты хранятся в памяти — не сохраняются между перезапусками. Шаблон по умолчанию регистрируется заново при старте.
5. **Экспорт JSON**: не восстанавливает загруженные файлы (Excel/шаблон) — только конфигурацию полей.
