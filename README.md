# Генератор сертификатов (Локальный веб-проект)

Локальное веб-приложение для автоматической генерации PDF-сертификатов по шаблону и данным из Excel-таблицы.

## Возможности

1. **Загрузка Excel (.xlsx, .xls)**: считывает данные, показывает колонки и первые 10 строк таблицы для предпросмотра.
2. **Загрузка шаблонов (PNG, JPG, PDF)**: использует первую страницу PDF или изображение как фон сертификата.
3. **Визуальный drag-and-drop редактор полей**:
   - Перемещение и изменение размеров полей мышью.
   - Точная ручная настройка координат (X, Y, ширина, высота).
   - Выравнивание текста (по горизонтали и вертикали), поворот, размер, цвет.
   - 4 режима переполнения: автоуменьшение шрифта (shrink-to-fit), многострочный перенос (multiline), обрезка (clip), в одну строку.
4. **Кастомные шрифты**: загрузка собственных `.ttf` / `.otf` файлов (необходимо для поддержки кириллицы).
5. **Импорт/Экспорт конфигурации**: сохранение настроек полей в JSON файл.
6. **Два режима экспорта**:
   - Раздельные PDF-файлы по шаблону имени (например, `{name}_{certificate_number}.pdf`).
   - Один объединенный PDF со всеми сертификатами постранично.
7. **Предпросмотр**: отображение тестовых данных в реальном времени, кнопка "Тестовый PDF" для мгновенной проверки.

## Быстрый запуск

### Шаг 1. Установка зависимостей

Откройте консоль в корневой папке проекта (`certificate-generator`) и выполните:

```bash
# Установка concurrently в корне, а также зависимостей фронтенда и бэкенда
npm install
npm run install:all
```

### Шаг 2. Генерация тестовых данных (опционально)

Для создания файлов примеров запустите скрипт:

```bash
node generate-sample.js
```
Скрипт создаст папку `sample-data/` с файлом `sample.xlsx` и файлом разметки `sample-config.json`.

### Шаг 3. Запуск серверов в режиме разработчика

Запустите оба сервера (бэкенд на порту 3001, фронтенд на порту 5173) одной командой:

```bash
npm run dev
```

После этого откройте в браузере: **http://localhost:5173**

## Руководство пользователя (Пошагово)

1. **Загрузка данных**: 
   - Загрузите Excel-таблицу (`sample-data/sample.xlsx`). Вы увидите список колонок и предпросмотр строк.
2. **Загрузка шаблона**: 
   - Загрузите изображение шаблона.
3. **Настройка шрифта (Кириллица)**:
   - Стандартные шрифты PDF (Helvetica, Times Roman) не поддерживают кириллицу.
   - Нажмите **"Загрузить шрифт (TTF/OTF)"** в шапке и выберите любой TTF-шрифт с поддержкой кириллицы с вашего компьютера (например, `Arial.ttf` или `Inter-Regular.ttf` из папки `C:\Windows\Fonts`).
4. **Конфигурация полей**:
   - Нажмите **"Импорт JSON"** в шапке и выберите `sample-data/sample-config.json` для автоматической настройки, ИЛИ
   - Нажмите **"+ Добавить поле"** слева, привяжите к колонке Excel, настройте положение и выберите ваш загруженный шрифт на панели справа.
5. **Экспорт**:
   - Выберите режим (раздельные или общий файл).
   - Нажмите **"Тестовый PDF"** для проверки текущей строки.
   - Нажмите **"Создать сертификаты"**. Готовые файлы сохранятся в папку `output/export_DATE_TIME/` в корне проекта. Вы также сможете скачать их архивом прямо из браузера.

---

## Известные ограничения

- **Кириллица**: Для отображения русских символов **обязательно** загрузите кастомный шрифт TTF/OTF. Стандартные шрифты PDF (Helvetica/Courier) выведут пустые символы или знаки вопроса.
- **Пути**: Не допускается сохранение файлов в системные папки (например, `C:\Windows`). По умолчанию используется папка `output/` в корне проекта.
